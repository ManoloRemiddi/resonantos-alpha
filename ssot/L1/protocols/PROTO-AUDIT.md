# PROTO-AUDIT — Preliminary Project Audit (Bridge Mode)

> **Level:** L1 (Protocol)
> **Created:** 2026-03-14
> **Trigger:** Manolo receives a project/text from a community member for feedback
> **Status:** Active

---

## Purpose

Provide rigorous, structural analysis of an external project. Output is addressed to Manolo (not the builder) for honest assessment without performative softening. Constructive, not destructive — build bridges from generic thinking to sovereign thinking.

## Trigger

Manual. Manolo shares a project/document and requests an audit.

## Constraints

1. **Address Manolo.** Write as a report to him. ("Manolo, I've reviewed the submission...")
2. **Self-Deception Check.** Primary metric: internal consistency. Are they building what they claim? Or using "liberation" language to build a capture mechanism?
3. **Respect their world.** Their framework may differ from ours. Don't force our terminology. Test if *their* world is internally stable.
4. **Bridge, don't burn.** If generic, don't say "this is bad." Say: "This is currently Tier 1. To reach Tier 2, the Builder needs to..."
5. **Capture Mechanism Scan.** Explicitly check: does this system, as designed, extract more value from participants than it returns?

## Tier Classification

| Tier | Label | Description |
|------|-------|-------------|
| 0 | Incoherent | Can't be audited meaningfully — lacks structure |
| 1 | Generic | Corporate/template thinking, possibly with liberation language on top |
| 2 | Sovereign | Unique insight, internally consistent, defensible |
| 3 | Resonant | Novel contribution to the field — advances the conversation |

## Output Format (Strict)

### 1. SOVEREIGN SCORECARD

Rate each dimension: **High / Medium / Low / Critical**

| Dimension | Score | Notes |
|-----------|-------|-------|
| **Clarity of Vision** | | Is the "Why" defined? |
| **Structural Integrity** | | Does the "How" match the "Why"? |
| **Sovereignty Factor** | | Unique insight or generic mimicry? |
| **Self-Deception Risk** | | Are they lying to themselves about the outcome? |
| **Signal-to-Noise Ratio** | | What % of the text is substance vs. filler/marketing? |
| **Overall Tier** | | 0–3 (see classification above) |

### 2. THE DIAGNOSIS (One Sentence)

One sentence. What this project *actually is*, stripped of marketing and emotional attachment.

> Example: "The Builder believes they are designing a community garden, but the incentives are structured like a data-mining operation."

### 3. THE BRIDGE (Constructive Friction)

Don't just list problems — offer pivots.

- **The Plausibility Trap:** Where does the text sound "professional" but lack deep meaning?
- **The Velocity Drift:** Where are they rushing to scale before they've defined?
- **The Capture Check:** Does the mechanism serve the participants, or extract from them?
- **The Pivot:** 1–2 concrete, high-leverage questions or changes that would instantly increase the resolution of this project.

### 4. SUMMARY FOR THE BUILDER

Two sentences Manolo can quote directly. Structural, not emotional.

> Example: "The mechanism is solid, but the philosophy is fragile. Reinforce the 'Why' before building the 'How'."
