## Decision-Making Levels

### Minor Changes
**Examples:** Bug fixes, documentation updates, small improvements

**Process:**
- Maintainer approval
- No DAO vote required
- Fast turnaround

### Medium Changes
**Examples:** New features, architectural changes, significant refactors

**Process:**
1. Community discussion (GitHub Discussions or Discord #roadmap)
2. Maintainer proposal
3. Community feedback period (3-7 days)
4. Maintainer approval with community input
5. Implementation

### Major Changes
**Examples:** Philosophy changes, license modifications, core architecture rewrites

**Process:**
1. DAO proposal submitted (requires $RCT tokens)
2. Discussion period (7 days minimum)
3. Voting period (7 days)
4. **67% approval threshold** required
5. Implementation after approval

## Earning Governance Rights ($RCT)

Contribute to earn soulbound governance tokens:

### Code Contributions
- **Merged PRs:** Weighted by impact and complexity
- **Bug fixes:** Small allocation
- **Major features:** Significant allocation
- **Architecture work:** High allocation

### Documentation
- **Tutorials:** Medium allocation
- **Guides:** Medium allocation
- **Translations:** Small-medium allocation
- **README improvements:** Small allocation

### Community
- **Moderation:** Regular allocation for active moderators
- **Support:** Helping others in Discord/forums
- **Bug reports:** Small allocation for verified reports
- **Testing:** Alpha/beta testing participation

### Protocol Creation
- **Protocol NFTs:** Earn $RCT for creating valuable protocols
- **Protocol Store:** Sales generate $RCT for creators
- **Community adoption:** Bonus $RCT for widely-used protocols

## Voting Process

### Proposal Submission
1. Hold minimum $RCT balance (threshold TBD)
2. Submit proposal with clear description
3. Include rationale and expected impact
4. Tag as `governance` in GitHub Discussions

### Discussion Period
- 7 days minimum for major proposals
- Community reviews and debates
- Proposer responds to questions
- Amendments can be made

### Voting Period
- 7 days for major proposals
- 3 days for medium proposals
- On-chain voting via Solana
- One $RCT = one vote
- Quadratic voting under consideration

### Execution
- Approved proposals move to implementation
- Maintainers coordinate execution
- Progress tracked publicly
- Community updates provided

## Maintainer Roles

### Core Maintainers
- **Current:** Manolo Remiddi
- **Authority:** Merge rights, release management, security decisions
- **Selection:** DAO vote for new maintainers (67% threshold)
- **Removal:** DAO vote with 75% threshold

### Component Maintainers
- **Scope:** Specific components (R-Memory, Dashboard, etc.)
- **Authority:** Merge rights for their component
- **Selection:** Core maintainer nomination + DAO approval (simple majority)

### Community Moderators
- **Scope:** Discord, forums, GitHub discussions
- **Authority:** Enforce Code of Conduct, manage community
- **Selection:** Core maintainer nomination + community input

## Treasury Management

### Funding Sources
- **1% Tithe:** Commercial users (>$10K/year revenue) contribute 1%
- **Protocol Store:** Transaction fees
- **Donations:** Community contributions
- **Grants:** External funding

### Fund Allocation
Approved by DAO vote:
- **Development (40%):** Core team, bounties, grants
- **Community (30%):** Events, documentation, education
- **Operations (20%):** Infrastructure, hosting, services
- **Reserve (10%):** Emergency fund

### Transparency
- All transactions on-chain (Solana)
- Quarterly treasury reports
- Public wallet addresses
- DAO vote for expenditures >$5K

## Conflict Resolution

### Process
1. **Discussion:** Attempt to resolve through dialogue
2. **Mediation:** Core maintainer or moderator mediates
3. **Community Input:** Broader community weighs in
4. **DAO Vote:** Final decision if unresolved (simple majority)

### Appeals
- Appeals can be submitted to core maintainers
- Must include evidence and rationale
- Reviewed within 7 days
- DAO vote for final appeal (if needed)

## Amendment Process

This governance document can be amended through:
1. DAO proposal for governance changes
2. 14-day discussion + voting period (longer than standard)
3. **75% approval threshold** (higher than standard)
4. Implementation after approval
