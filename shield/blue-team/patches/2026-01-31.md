## Scanner.py Patches

### 1. Unicode Normalization Support
- Added `unicodedata` import
- Added `UNICODE_CONFUSABLES` mapping for circled letters, fullwidth chars, Cyrillic look-alikes
- Added `INVISIBLE_CHARS` list for zero-width and RTL override characters
- Added `_normalize_unicode()` method to sanitize text before scanning
- Modified `scan_input()` to scan both original and normalized text

### 2. Reversed Text Detection
- Added `_check_reversed_text()` method
- Checks both character-reversed and word-reversed text for injection keywords

### 3. Enhanced Direct Injection Patterns
- `direct_override`: Made pattern more flexible (`ignore all instructions` now matches)
- `direct_override_2`: Added "disregard your prior programming"
- `direct_override_3`: Added "forget everything you know/learned"
- `fake_system_hash`: Added "###SYSTEM###" format
- `fake_system_asterisk`: Added "**SYSTEM**" format
- `fake_system_equals`: Added "==SYSTEM==" format

### 4. Social Engineering Patterns (NEW)
Added new `SOCIAL_ENGINEERING_PATTERNS` group:
- `social_urgency`: Emergency/urgent/critical pretexts
- `social_authority`: Boss/manager pressure tactics
- `social_pressure`: "You must do this now" patterns
- `social_test_pretext`: "This is just a test" + sensitive data
- `social_dev_pretext`: "For debugging purposes"
- `social_hypothetical`: Hypothetical bypass scenarios
- `social_pretend`: "Pretend you're not an AI"
- `social_trust`: Trust-building manipulation
- `social_colleague`: "We're colleagues" social engineering

### 5. Enhanced Exfiltration Patterns
- `exfil_ssh_key`: Extended to match "contents of ~/.ssh/id_rsa"
- `exfil_ssh_path`: Direct SSH key path detection
- `exfil_api_key`: Extended to match "What is your API key"
- `exfil_api_key_query`: Direct API key query pattern
- `exfil_vendor_key`: Specific vendor key names (ANTHROPIC, OPENAI, etc.)

### 6. Enhanced RCE Patterns
- `rce_os_system`: Detect `os.system()` calls
- `rce_subprocess`: Detect `subprocess.call/run/Popen` calls
- `rce_eval`: Extended to match all `eval()` patterns
- `rce_exec`: Detect `exec()` calls
- `rce_import_os`: Detect `__import__('os')`
- `rce_compile_exec`: Detect compile+exec patterns
- `shell_command_injection`: Detect `; cat|ls|id|whoami`
- `command_substitution`: Detect `$()` and backtick patterns

### 7. XML Entity & Nested Encoding Detection
- Added `_decode_xml_entities()` method
- Added `_recursive_decode()` for nested base64/hex
- `encoded_injection_xml_entity`: Detect XML entity encoded payloads
- `encoded_injection_nested`: Detect recursively encoded payloads

## A2A Monitor Patches

### 1. New Threat Types
Added new `A2AThreatType` enum values:
- `TYPOSQUATTING`
- `NESTED_TASK_INJECTION`
- `RESPONSE_INJECTION`
- `FIELD_INJECTION`

### 2. Typosquatting Detection
- Added `LEGITIMATE_DOMAINS` mapping with known typosquat variants
- Added `SUSPICIOUS_TLDS` list (`.xyz`, `.top`, `.tk`, etc.)
- Added `_check_typosquatting()` method with:
  - Known variant matching
  - Suspicious TLD + brand name detection
  - Similarity threshold checking
- Added `_is_similar()` for Levenshtein-like comparison

### 3. Comprehensive Field Scanning
- Added `_scan_all_fields()` method that scans:
  - name, description, url, version
  - All skills
  - All permissions
  - Recursively flattened metadata

### 4. Nested Task Detection
- Added `_check_nested_tasks()` method
- Recursively checks payload for task-like structures
- Detects injection attempts in nested `task`, `subtask`, `command`, `instruction` keys

### 5. Response Injection Checking
- Added `check_response()` method for scanning A2A responses

### 6. Enhanced verify_agent_card()
- Now checks typosquatting BEFORE trusted patterns
- Runs comprehensive field scanning including metadata
- Blocks cards with injection in any field
